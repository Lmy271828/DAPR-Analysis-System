# DAPR 知识库修改指南

> 本知识库直接影响 DAPR-Analysis-System 对用户的分析输出。任何修改都必须经过伦理一致性审查，确保与系统的"去临床化、艺术引导"定位保持一致。

## 修改流程

```
┌─────────────┐    ┌──────────────────┐    ┌──────────────┐    ┌─────────────┐
│  1. 编辑文档 │ → │ 2. 本地伦理检查   │ → │ 3. 人工评审   │ → │ 4. 提交合并  │
└─────────────┘    └──────────────────┘    └──────────────┘    └─────────────┘
```

### 1. 编辑文档

- 所有知识库文档使用 **Markdown** 格式
- 文档编号按主题顺序（01-07），新增主题需讨论编号
- 引用外部文献时必须标注来源链接或 DOI
- 使用中性语言，避免病理化术语

### 2. 本地伦理检查（自动化）

在提交前，**必须**运行伦理检查脚本：

```bash
# 从项目根目录运行
cd /path/to/DAPR-agent
python scripts/ethics_check.py

# 或检查指定文件
python scripts/ethics_check.py docs/dapr_knowledge_base/05_ethical_guardrails.md
```

检查规则：

| 级别 | 说明 | 处理方式 |
|------|------|---------|
| ❌ **Error** | 病理化术语、确定性断言、评分量化、伪科学表述 | **必须修复**，阻断提交 |
| ⚠️ **Warning** | 敏感表述（如"防御机制弱""自我力量低"）| 需人工复核，确认上下文恰当后可放行 |

### 3. 人工评审清单

即使自动化检查通过，仍需人工确认以下事项：

- [ ] **定位一致性**：新增内容是否支持"艺术表达引导"定位，而非悄悄滑向"心理评估"
- [ ] **文化谦逊**：是否隐含了某一文化的标准作为"正常"基准？
- [ ] **年龄中立**：是否将发展性特征误读为心理状态？
- [ ] **确定性审查**：是否包含"X 意味着 Y"的确定性表述？应改为"X 可能与 Y 有关，但也可能是..."
- [ ] **来源可验证**：引用的研究、数据是否有明确出处？
- [ ] **prompt 影响评估**：该修改被注入系统提示后，是否可能导致对用户的误导？

### 4. 提交规范

```bash
# 知识库修改必须单独提交，不与代码变更混在同一个 commit
git add docs/dapr_knowledge_base/
git commit -m "kb: 更新文化适应性数据（添加泰国微笑研究）

- 新增泰国 9-17 岁儿童微笑面孔比例数据
- 补充 Gardiner 跨文化研究来源
- 通过 ethics_check.py 检查
- 人工评审：确认无病理化表述"
```

Commit message 前缀：
- `kb:` — 知识库内容更新
- `kb-fix:` — 知识库错误修正
- `kb-ethics:` — 伦理护栏相关修改

## 禁止事项

以下类型的内容**不得**加入知识库：

1. **诊断标准**：任何用于做出精神疾病诊断的标准或量表
2. **治疗建议**：任何具体的心理治疗或干预方案
3. **确定性的文化刻板印象**：如"中国人都..." "日本人总是..."
4. **未经同行评审的研究**：预印本、自媒体文章、营销材料
5. **商业产品推广**：任何心理治疗产品或服务的推广内容

## 允许事项

以下内容**欢迎**加入：

1. 经同行评审的学术论文摘要
2. 艺术治疗领域的最佳实践指南
3. 跨文化研究的基线数据
4. 投射测验效度争议的学术讨论
5. AI 辅助分析的前沿研究综述

## 紧急修改通道

如果发现知识库中存在**严重伦理风险**（如可能导致用户受到心理伤害的内容），可跳过常规流程直接修复，但必须在 24 小时内补全文档和评审记录。

紧急修复的 commit message 格式：
```
kb-ethics(hotfix): 移除 [具体问题] 的误导性表述

风险：[简述风险]
修复：[简述修复]
来源：[相关文献或讨论]
```
