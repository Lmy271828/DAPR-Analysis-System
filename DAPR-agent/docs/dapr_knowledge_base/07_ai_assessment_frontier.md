# AI 辅助绘画分析前沿

> 了解 DAPR 领域内 AI 辅助分析的技术标杆和最佳实践，帮助系统理解当前技术边界和负责任的使用方式。

## 主要研究项目

### AlphaDAPR（2023, ACM IUI）

- **全称**：An AI-based Explainable Expert Support System for Art Therapy
- **研究者**：Jiwon Kim 等，韩国 DSAIL-SKKU 团队
- **技术**：YOLOv5 目标检测 + 可解释 AI
- **功能**：
  - 将 DAPR 绘画对象分为**压力相关对象**（雨、云）和**资源相关对象**（雨伞、雨衣）
  - 自动识别 6 类主要物体：人物、雨、伞、闪电、积水、云
  - 自动计算 DAPR 分数
- **数据集**：330 张人类绘制场景草图 + 3,300 张合成草图
- **临床应用**：支持艺术治疗师和心理学家进行大规模自动绘画评估
- **治疗师接受度**：大多数艺术治疗师表现出高使用意向，同时表达对 AI 局限性和威胁的担忧
- **来源**：https://dl.acm.org/doi/abs/10.1145/3581641.3584087

---

### SceneDAPR（2024, ACM WWW）

- **全称**：A Scene-Level Free-Hand Drawing Dataset for Web-based Psychological Drawing Assessment
- **研究者**：Jiwon Kang 等，韩国 DSAIL-SKKU 团队
- **数据集规模**：
  - **1,399 张场景级手绘**
  - **6,420 个标注物体**
  - 年龄分布：儿童青少年 8-18 岁（462张）、成人 19-59 岁（650张）、老年人 60-96 岁（287张）
- **技术**：YOLO-v8 目标检测的网页版评估系统
- **验证结果**：
  - 专家间 ICC(2,k) = 0.94-0.98（高度一致）
  - 模型与专家 ICC(3,1) = 0.75-0.77（良好一致，≥0.75）
- **关键发现**：
  - 排除老年人数据训练的模型在识别老年人绘画时表现较差
  - 凸显**年龄多样性**在训练数据中的重要性
- **GitHub**：https://github.com/DSAIL-SKKU/SceneDAPR
- **来源**：https://dl.acm.org/doi/10.1145/3589334.3648150

---

### CheckDAPR（2025, ACM CIKM）

- **全称**：An MLLM-based Sketch Analysis System for Art Therapy
- **研究者**：DSAIL-SKKU 团队
- **技术突破**：**YOLO-v8 目标检测 + Claude 3.5 Sonnet（多模态大语言模型 MLLM）**
- **两阶段流程**：
  1. YOLO-v8 检测绘画中的对象
  2. MLLM 精炼检测结果，纠正素描中的不准确识别
  3. 生成 DAPR 评估报告，计算详细分数
- **效果**：显著减少艺术治疗师的分析时间
- **GitHub**：https://github.com/DSAIL-SKKU/CheckDAPR
- **来源**：https://dl.acm.org/doi/10.1145/3746252.3761571

### 中国研究：Li 等（2021）DAPR 图像分析软件

- 开发软件自动捕捉 DAPR 绘画图像
- 准确读取：**雨量、雨滴间距、雨滴长度、雨覆盖面积**
- 与中国压力反应量表呈**中度相关**
- 民航飞行员研究（2024）采用 6 项评分，发现飞行员压力应对资源不理想

## 数字化评估的优势与局限

### 优势

| 优势 | 说明 |
|------|------|
| 消除评估者间差异 | 标准化检测减少主观偏差 |
| 过程数据获取 | 可测量绘画速度、笔画数、笔画长度变化等传统评估无法获取的数据 |
| 用户偏好 | 受试者对绘画评估的主观偏好高于问卷（抑郁者偏好更强）|
| 大规模筛查 | 适合学校、工作场所的初筛 |
| 减少专家时间 | 减少艺术治疗师的分析时间和成本 |

### 局限

| 局限 | 说明 |
|------|------|
| 信息有限 | 绘画仅揭示部分心理状态信息，不足以单独作为筛查工具 |
| 贡献度低 | 与人口统计学信息相比，绘画特征的贡献度较低 |
| 数据缺乏 | 缺乏大规模、专业环境收集的标注数据集 |
| 可解释性 | AI 决策过程需要可解释性，用户和治疗师需要理解为什么 |
| 定制化需求 | 需要考虑治疗师个体实践差异的定制化需求 |

## 对本系统的启示

### 技术定位

本系统（DAPR-Analysis-System）的技术路线与 CheckDAPR 最为接近：

| 维度 | CheckDAPR | 本系统 |
|------|-----------|--------|
| 目标检测 | YOLO-v8（本地）| Qwen3.5 VLM（本地，多模态理解）|
| 语义理解 | Claude 3.5（API）| Kimi-K2.5（API，文本）+ Qwen3.5（本地，多模态）|
| 评分计算 | 是 | **否**（去临床化定位）|
| 可解释性 | 提供 | 通过温和叙述提供 |
| 治疗师参与 | 辅助系统 | **用户直接面对的系统**（无治疗师在场）|

### 关键差异

CheckDAPR 定位为**治疗师辅助工具**——治疗师在场，AI 提供参考，最终判断由人类做出。

本系统定位为**用户直接面对的艺术探索伙伴**——无治疗师在场，系统输出的每一句话都直接影响用户。这要求系统的**伦理护栏必须比 CheckDAPR 更严格**。

### 可借鉴的最佳实践

1. **年龄多样性**：SceneDAPR 发现排除老年人数据会导致年龄偏见。本系统应特别注意对老年用户和儿童用户的分析校准。
2. **可解释性**：AlphaDAPR 强调可解释 AI。本系统通过温和叙述提供解释，而非黑盒评分。
3. **两阶段分析**：CheckDAPR 的"检测 + 精炼"流程可借鉴——本系统可先由 VLM 做视觉理解，再由文本 LLM 做温和叙述生成。
4. **治疗师担忧**：AlphaDAPR 的研究者发现治疗师对 AI 局限性和威胁的担忧。本系统应始终明确自身定位，不替代专业评估。
